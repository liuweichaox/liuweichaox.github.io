01 · Why Ingot

遥测是原料，
生产事实才是结果。

设备信号数量大、频率高，却不天然具备业务含义。Ingot 在边缘侧把原始变化标准化、事件化，让生产过程能够被查询、追踪和解释。

原始数据像矿砂。Ingot 用统一模型把它铸成可堆叠、可追溯的“锭”。

项目以 Edge Agent 为核心运行组件。PLC 是当前首个源适配器；遥测与生产事件采用不同的数据语义与存储路径，使实时采集和可靠事实能够分别演进。

03 · Capabilities

围绕真实工业场景，
构建完整的数据链路。

能力重点不是单个功能，而是采集、事实成立、失败恢复、中心汇聚和可观测性之间的整体一致性。

01

配置化数据源

通过 JSON 配置管理数据源、资产、遥测通道和事件规则，并支持配置校验与热更新。

02

双采集模式

支持 Always 与 Conditional 两类采集模式，在持续遥测与条件触发之间进行明确选择。

03

不可变事件日志

生产事件先写入 SQLite 本地日志，落盘即视为事实成立，后续投影可以重建。

04

可靠上行

待上行事件按序批量发送到 Central，失败时指数退避，确认后才推进本地 outbox。

05

查询与实时流

提供事件、周期和上下文查询，并通过 SSE 暴露边缘与中心侧的实时事件流。

06

可观测运行时

将指标、日志、配置诊断和模拟器纳入系统，使问题能够被复现、定位和验证。

04 · System semantics

可靠性不是补丁，
而是系统语义。

Ingot 明确区分遥测平面和事件平面，以不同的交付保证处理不同价值密度的数据。

Core boundary

Edge Agent 是核心，Central 是可选控制面。

中心系统不可用时，边缘采集和事件产生仍可继续。恢复连接后，事件从本地 outbox 按序续传并在中心进行幂等处理。

01

遥测平面

采用 at-most-once 语义。TSDB 写入失败时记录错误并丢弃当前批次，避免阻塞采集链路。

02

事件平面

以 at-least-once 为基础。事件先落盘，再通过 outbox 可靠上行到中心事件库。

03

中心幂等

中心使用 EventId 与 EdgeId、Seq 组合键去重，防止重试造成重复事实。

04

可重建投影

事件日志是事实来源，时序和查询投影可以从日志重新生成。
